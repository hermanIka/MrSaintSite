2️⃣ NOUVELLE ARCHITECTURE DE PAIEMENT (OBLIGATOIRE)

- PowerPay → Mobile Money
- LemonSqueezy → Carte bancaire
- PayPal → Paiement PayPal

Chaque provider doit :

4️⃣ POWERPAY (MOBILE MONEY)

- Préparer l’intégration PowerPay :
  - aucune clé API réelle codée en dur
  - utiliser des variables d’environnement :
    POWERPAY_API_KEY
    POWERPAY_BASE_URL
- Implémenter :
  - initialisation d’un paiement Mobile Money
  - gestion du statut (pending / success / failed)
  - webhook prêt à recevoir la confirmation PowerPay
- Ne PAS simuler le paiement côté client.
- Tout paiement doit passer par le backend.

5️⃣ LEMONSQUEEZY (CARTE BANCAIRE)

- Préparer l’intégration LemonSqueezy :

    LEMONSQUEEZY_API_KEY
    LEMONSQUEEZY_STORE_ID
    LEMONSQUEEZY_WEBHOOK_SECRET
- Implémenter :
  - création de checkout LemonSqueezy
  - redirection utilisateur vers LemonSqueezy
  - gestion du retour de paiement
  - webhook de validation
- Le frontend ne fait QUE déclencher le paiement.

6️⃣ PAYPAL
