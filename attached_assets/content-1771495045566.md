# Maishapay REST API DOC

**MaishaPay REST API Overview**

MaishaPay REST API is a powerful and flexible solution that allows businesses, merchants, and developers to integrate advanced payment and financial transaction management features into their existing applications or systems.

### Objectives of MaishaPay REST API

MaishaPay REST API provides a suite of tools for:

**Online Collection:** Enabling merchants to collect online payments via various solutions, including:

- **Mobile Money:** Integration with mobile wallets like Airtel Money, M-Pesa, and Orange Money.

- **Bank Cards:** Support for payments via Visa, Mastercard, and other cards.

- **Cryptocurrencies:** Integration for receiving payments in crypto.


**Fund Transfers:**

- Direct transfer to Mobile Money accounts.

- Transfer to bank cards.


**Mobile Recharge and Data Sales:**

- Sale of phone credits and mobile data available in over 75 countries.

**TV Subscription Payments:**

- Limited support for TV subscriptions in the Democratic Republic of Congo (DRC).

**International Operations:** The platform operates in several African countries, including:

- Democratic Republic of Congo (DRC)

- Republic of Congo

- Cameroon

- Senegal

- Benin

- Ivory Coast

- Togo

- And soon in Ghana, Nigeria, Egypt, etc...


**Partnership with CyberSource (Visa):**

For card payments, MaishaPay relies on CyberSource, a secure and robust payment platform provided by Visa.

### Supported HTTP Methods

- **GET:** Retrieve data.

- **POST:** Send or create new data.

- **PUT:** Update existing data.

- **DELETE:** Remove resources.


### Response Structure

The API responses are uniform and in JSON format.

### Common Use Cases

- Collect payments for products or services.

## Collection

The MaishaPay Collection module allows merchants to receive payments directly through three main channels: Mobile Money, Bank Cards, and Cryptocurrencies. This module provides a fast, secure, and integrated solution to simplify online merchants' financial transactions.

The collected funds can be transferred to Mobile Money wallets or bank accounts, offering great flexibility to users.

## Collection via Mobile Money

Mobile Money collection is a key feature of MaishaPay that allows merchants to receive payments directly through Mobile Money operators available in several African countries. This system facilitates financial transactions in environments where electronic wallets are widely used, providing an accessible and convenient solution for end users.

### Special Case: PENDING (201) Transaction

**Description:**

It may rarely happen that a transaction remains in a PENDING (201) status due to a delay or technical issue with the operator (M-Pesa).

**Impact:**

The final customer may be charged, but the transaction will not be immediately confirmed by the operator.

In this case, the merchant or the final customer is asked to provide proof of debit (e.g., a screenshot or statement) to allow further verification with the operator.

**Example of Response for Pending Transaction**

json

```json
{
    "status": 201,
    "data": {
        "transactionReference": "gdt458",
        "statusCode": "202",
        "status": "PENDING",
        "transactionDate": "01/01/2023 01:01:01",
        "transactionDescription": "Transaction awaiting confirmation from the operator.",
        "transactionId": "123456"
    }
}
```

### Procedure for PENDING Transaction

- A notification is sent to the merchant indicating that the transaction is pending.

- The merchant or final customer must provide proof of debit (e.g., a screenshot).

- MaishaPay uses this proof to perform a verification with the operator.

- Once the verification is complete, MaishaPay sends an update to the merchant with the final status of the transaction (APPROVED or DECLINED).


### Validation Process

- Upon sending the request, a push notification is sent to the final customer's phone.

- The customer must validate the transaction by entering their PIN code on their phone, ensuring that only the account holder can authorize the transaction.

- Once the validation is complete, the final status of the transaction is transmitted to MaishaPay and sent back to the merchant via the same connection.


### Conclusion

In this version, there is no callback sent to the merchant. The connection between the merchant's server and MaishaPay's server remains active until the final transaction status is received from the M-Pesa operator. This method ensures an instant and secure response for each transaction.

This modular system offers merchants the flexibility to choose the version that best suits their needs in terms of user experience and integration.
